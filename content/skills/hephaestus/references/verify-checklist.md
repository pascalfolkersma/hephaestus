# Verify checklist

Structural verification checks run by `hephaestus` after `npx @pascalfolkersma/hephaestus init --config init.yaml` completes. Checks are listed in the order they are executed in SKILL.md Step 6. For each check, the Auto-fix column describes what `hephaestus` does autonomously; the Report-only column describes what `hephaestus` surfaces to the user without acting.

Auto-fix scope is bounded to **deterministic structural discrepancies** — cases where the correct state is unambiguous and the fix cannot destroy user content. Content-level and judgment-level discrepancies are reported only.

---

## Check table

| # | Check | What to look for | Auto-fix action | Report-only action | Trigger condition |
|---|---|---|---|---|---|
| 1 | **Expected directories exist** — `.claude/`, `.claude/agents/`, `.claude/skills/`, `.claude/memory/`, `.claude/hooks/` are present at `<targetDir>`. | Any of the five directories is absent or does not exist as a path. | Create the missing directory and write a `.gitkeep` inside it. | — | One or more directories missing after init. |
| 2 | **Expected files exist** — `AGENTS.md` is present at `<targetDir>`; `.claude/settings.json` is present. | Either file is absent. These are generated by init, so absence signals an init error. | — | Report each missing file by name; note that absence likely signals an init failure; suggest re-running init. | One or both files absent after init. |
| 3 | **`.gitignore` entries present** — `/dist` and `.claude/flows/` are each present as a line in `<targetDir>/.gitignore`. | Any of the two required entries is absent from `.gitignore`, or `.gitignore` itself does not exist. | Append the missing entry (or entries) to the end of `.gitignore`. If the file does not exist, create it. Never remove or reorder existing entries. | — | One or more required entries missing; or `.gitignore` absent. |
| 4 | **No flat-copy artifacts** — No skill files (`SKILL.md`, `UPSTREAM.md`, or a skill-named `README.md`) exist directly at `<targetDir>` root or directly under `<targetDir>/.claude/` (not inside a named subdirectory under `.claude/skills/`). | `<targetDir>/SKILL.md`, `<targetDir>/UPSTREAM.md`, `<targetDir>/.claude/SKILL.md`, `<targetDir>/.claude/UPSTREAM.md`; `<targetDir>/README.md` if its first line is a known Hephaestus skill heading. | If the artifact's destination directory `<targetDir>/.claude/skills/<name>/` is empty or absent: move the artifact there (write to new path, delete original). Create destination directory if needed. | If destination is non-empty or skill name cannot be determined from the artifact: report the artifact path and the ambiguity; do not move. | Skill folder was copied with wrong destination path during init or manual setup. |
| 5 | **Skill folders intact under `.claude/skills/`** — Any skill expected to be installed has a non-empty folder (more than just `.gitkeep`) under `<targetDir>/.claude/skills/`. The `lore-keeper` skill is always expected. | A skill folder is absent, or exists but contains only `.gitkeep`. | If the missing skill is present in the hephaestus bundle (currently: `lore-keeper`), copy the full skill folder from `.claude/skills/hephaestus/<skill-name>/` to `<targetDir>/.claude/skills/<skill-name>/`. | If the missing skill is not in the hephaestus bundle: report the skill name with "not in hephaestus bundle — re-run init to restore". | Skill folder absent or empty after init; or flat-copy artifact was removed without restoring proper folder. |
| 6 | **Agent frontmatter valid** — Each `.md` file under `<targetDir>/.claude/agents/` has a valid YAML frontmatter block (between `---` delimiters) containing at minimum `name:` and `description:` fields. | Frontmatter delimiters missing; `name:` field absent; `description:` field absent; YAML between delimiters has a parse error (duplicate keys, invalid indentation). | — | Report each agent file with a frontmatter issue: include the file path and the specific problem (missing field, parse error message). |  Any agent file is malformed or incomplete. |
| 7 | **Hooks syntactically runnable** — Each `.js` file under `<targetDir>/.claude/hooks/` passes `node --check`. | `node --check <file>` exits non-zero for any hook file. | — | Report the file path and the full error output from `node --check`. If no `.js` files exist under `.claude/hooks/`, skip this check (not a failure — note as skipped). | A hook file was written with a syntax error, or a template substitution left invalid JavaScript. |
| 8 | **Contract-validator passes for rendered agents** — If `core/lib/validator.js` is present at `<targetDir>`, run it against `<targetDir>/.claude/agents/` and report its output. Falls back to a search for any `validator.js` under `core/` if the primary path is absent. | Validator exits non-zero; or validator is absent (skip, not failure). | — | Report the full validator output if exit code is non-zero. If no validator is found, note "contract-validator not found — check skipped". | Rendered agents have structural or schema problems not caught by init itself. |

## Scope boundary reminder

The checks above are the complete v1 scope for the verify-and-fix boundary.

**Not checked (out of scope for v1):**
- Roadmap-alignment (do the agents match the project's roadmap entries?)
- Agent content quality (are descriptions accurate, are tool lists appropriate?)
- Semantic correctness of AGENTS.md sections
- Consistency between AGENTS.md and `.claude/agents/` content

These are judgment-level checks. A future version of `hephaestus` may add report-only heuristics here. For v1, `hephaestus` does not touch them.
