# Changelog

All notable version releases for `@pascalfolkersma/hephaestus`.

## 0.20.x — Interactive harness selection + init upgrade hardening + MIT license

- **0.20.1** — **Re-publish from the new public `hephaestus` repository:** no functional changes. Development moved to a fresh public repository (`pascalfolkersma/hephaestus`) with clean history; the prior workshop repository is archived. This patch re-establishes the GitHub Packages publish pipeline from the new repository — `0.20.0` remains unchanged in the registry. No source, agent, skill, or template behaviour is affected.
- **0.20.0** — **Interactive harness selection in Phase 1 install (ADR 0047 / Decision 0045):** `npx @pascalfolkersma/hephaestus install` now opens an interactive prompt in TTY contexts so the user can confirm or override the auto-detected target harness (`claude-code` / `copilot`). The detected harness is offered as the pre-selected default — an Enter keypress accepts it; typing an alternative overrides it. A four-level precedence chain governs harness resolution: `--harness=<value>` CLI flag → `HEPHAESTUS_HARNESS` env var → interactive readline prompt (TTY) → silent detected-default fallback (non-TTY). Unknown non-empty values from the flag or env var exit with an error naming the bad value and the valid set; invalid typed values at the prompt fall back to the detected default with a log line. Non-TTY contexts and CI pipelines are fully unaffected — the silent detected-default fallback is the pre-amendment behavior unchanged. Retracts ADR 0036's "Phase 1 does not open a readline loop" clause. Implementation surface: `core/install.js` (precedence chain), reusing existing `core/lib/readline.js` / `core/lib/prompt.js` without modification. New test cases IN9–IN19 (`test/integration/install-subcommand.test.js`) and HP1–HP3 (new `test/unit/install-harness-prompt.test.js`).
- **0.20.0** — **License switch: PolyForm Internal Use License 1.0.0 → MIT License (Decision 0047):** root `LICENSE`, `package.json` `license` field, `README.md` License section, `scripts/build.js` license-sync comments, `content/skills/hephaestus/LICENSE` and `dist/` mirrors, and the license-consistency test (`test/distribution/license-consistency.test.js`) now all reflect the MIT License. The staged license ladder from Decision 0032 is retired; any use including commercial use, modification, and redistribution is permitted with attribution. Supersedes Decision 0043 (PolyForm Internal Use) and transitively the license-choice section of Decision 0031.
- **0.20.0** — **--config skill-gap detection (M9.73 / Decision 0046):** `npx hephaestus init --config init.yaml` no longer silently drops skills that were added to Hephaestus after the YAML was generated. A Surface A pre-flight in `core/init.js` computes `listAvailableSkills() ∖ configAnswers.skills` after loading the config file. On a TTY it opens an interactive merge prompt (adopt all / keep / pick individually); on non-TTY it emits a non-fatal stderr warning naming each dropped skill and continues with exit code 0. An empty gap (up-to-date YAML) produces no output or behaviour change. C7/C8 integration tests cover both paths.
- **0.20.0** — **`.bak` backups on upgrade for `dispatch-enforce.config.json` and `hephaestus-context.json`:** the conflict handler now writes `.bak` safety copies for these two files when an upgrade run overwrites them, consistent with the existing CLAUDE.md / AGENTS.md spine-file backup policy. Both files now appear in the `POST_INIT_ENRICH.md` pairing list so the LLM can merge any user edits in the Phase 9 enrichment session.
- **0.20.0** — **`subagent-tracker.js` copied by init:** `content/.claude-template/hooks/subagent-tracker.js` was referenced by the generated `settings-snippet.json` but was not actually written to target projects during init. The copy step is now wired; target projects receive the hook automatically on a fresh or upgrade init.
- **0.20.0** — **dispatch-enforce.js template investigated:** a reported regression (gate-parity gap between the self-hook in `scripts/hooks/` and the shipped template in `content/.claude-template/hooks/`) was investigated and found to be a non-issue — both copies are functionally equivalent for all gate-relevant paths. No change made.

## 0.19.x — Flow 6: Claude Design ingest

- **0.19.0** — **Flow 6: Claude Design ingest pipeline (ADR 0046 / Decision 0044 / M16):** a new canonical flow that ingests a Claude Design (`claude.ai/design`) project via the DesignSync / `claude_design` MCP connector and runs the full idea→artefacts→review→sync-check→commit pipeline. The flow materializes design files into the knowledge base at `<docs>/raw/design/<date>-<slug>/`, writes a provenance file alongside each ingested artifact, and wraps all untrusted content in an untrusted-content guard before any further processing. The dispatch-enforce gate now admits `flow:6` on both the Claude Code and Copilot adapters. New additions: `design-sync` skill and `design-ingest-helpers` module; Flow 6 documented in `flows.md` (lore + shipped template in lockstep). Also in this release: scope-keyword detection made English-only with an extended Dutch-leak guard (M11.22); remaining Dutch text in shipped files translated to English (M11.21); workshop-internal references stripped from all shipped content for pre-public hygiene.

## 0.18.x — License switch + init-flow UX hardening

- **0.18.0** — **License switch: PolyForm Strict 1.0.0 → PolyForm Internal Use License 1.0.0 (Decision 0043):** updates root `LICENSE`, `README.md`, `CLAUDE.md`, and the license-consistency test marker; the build auto-syncs `content/skills/hephaestus/LICENSE` and `dist/` from the root.
- **0.18.0** — **Init-flow UX hardening: `--dry-run`, `--show-diff`, and `output_language` discoverability (M9.14–M9.16):** `--dry-run` threads a `dryRun` flag through all conflict handlers and every write helper (including upgrade marker-merge paths in `agents-md.js` and `project-files.js`); a dry run completes the full introspection and conflict-detection pipeline without touching disk and prints a per-file written/skipped/overwritten report. `--show-diff` extends the conflict prompt with an in-file LCS unified diff (zero new npm deps; binary files fall through silently); without the flag, a one-liner change estimate `(would change ~N lines)` is shown instead. `output_language` prompt label updated to `Output language (sets the "All prose in X" convention in CLAUDE.md and AGENTS.md)` so agents and users understand the downstream effect before accepting the default; upgrade-mode prefill via `priorContext` was already intact and verified.

## 0.17.x — Flow re-orientation layer

- **0.17.0** — **Advisory flow re-orientation layer: `plan.json` + `where-am-i.md` per session directory (ADR 0045 / Decision 0042 / M8.34–M8.35):** the main thread now writes a `plan.json` (per-run plan with named steps and back-edges) alongside the existing `context.json` in the session directory; a new `SubagentStop` hook (`subagent-tracker.js`, byte-identical in `content/.claude-template/hooks/` and `scripts/hooks/`) advances the step pointer after each subagent exit and re-renders `where-am-i.md` so the main thread always knows where in the flow it stands. The layer is strictly advisory — no verdict adjudication, never gates a dispatch, fail-open — and is registered in both `.claude/settings.json` and `content/.claude-template/settings-snippet.json`. A re-orientation pull hint has been added to the orchestrator agent body, the project-context template, and `CLAUDE.md`. `plan.json` write convention and schema are documented across all five flows in `flows.md` (lore + shipped template, in lockstep), including bringing the shipped template's flow 1/4 reviewer→sync-check stage into lockstep with the lore copy. ADR 0041 §2 write-convention reconciliation completed. 23 new tests. Also in this release: Copilot subagent-dispatch vocabulary wired into the enforcement gate (M12.4); scope-gate tests (`CSG_M12.4`) made self-contained and CI-independent.

## 0.14.x — Release flow (flow 5)

- **0.14.0** — **Flow 5, the release flow (Decision 0038 / ADR 0044):** cutting a release is now a human-confirmed flow step rather than ad-hoc `npm publish` + manual version math. At the end of a build (flow 2) or bug-fix (flow 3) cycle, `git-commit-push` asks "Should a release be cut now?" (default N); on yes, the flow-5 sequence runs build → test → sync-check → conventional-commit version analysis → confirmation → `npm version <derived>` → `git push --follow-tags`, which triggers the `.github/workflows/publish.yml` CI publish. New modules: `core/lib/release-bump.js` (Conventional-Commits-compliant bump analyzer — `feat`→minor, `fix`/`refactor`/`perf`/`revert`→patch, `!`/`BREAKING CHANGE`/`BREAKING-CHANGE`→major, pre-1.0 breaking demoted to minor, case-insensitive types; no-prefix range defaults to patch with a warning) and `core/lib/flow5-sequence.js` (the orchestrated sequence with injectable seams and clean abort semantics). The dispatch-enforce flow-context gate now accepts `flow:5`. Flow 5 documented in `flows.md` (lore + shipped template). Also in this release: `package-lock.json` is now committed and the committed-`dist/` model is documented (Decision 0039 / M11.5), making the CI `npm ci` publish path viable; dispatch-enforce config path and context-store read-priority derived from the target-adapter `TARGETS` seam rather than hardcoded (M12.29 / ADR 0039); Copilot CLI dual-vocabulary tool recognition (`edit`/`editFiles`, `create`/`createFile`) and `version: 1` in `hooks.json` (M12.30 / ADR 0039 second amendment).

## 0.13.x — Copilot as a co-equal first-class target

- **0.13.0** — **Copilot adapter v2 (Decision 0036 / ADR 0039 Path C):** Copilot becomes a co-equal first-class target via a true target-adapter seam (`core/lib/target-adapter.js`). Every per-target surface now resolves through the adapter: paths (`.github/` vs `.claude/`), memory location (`.github/memory/`), hook-config schema (`timeout` not `timeoutSec`, PascalCase events), deny convention (Copilot = exit 0 + payload, never `exit(2)`), session-id source (`sessionId` from stdin), and subagent side-channel (`.github/.copilot-active-subagent`). Init branches per target — a `shell=copilot` install writes **zero `.claude/` files** (hooks, agents, memory, dispatch-config, context-store, and post-init markers all route to `.github/`); `shell=claude-code` and `shell=both` unchanged. ~85 new tests (adapter contract, per-target E2E tree, shell-gate, hooks.json schema). **Known limitation:** Copilot's subagent-dispatch tool name is an unverified placeholder pending a live VS Code Copilot Chat session (M12.12/M12.4/M12.15/M12.5 open); the flow/scope gates fail safe (inert, not broken) until confirmed. Also in this release: external URLs removed from shipped content (M12.16); post-init phases renumbered to match execution order — concept→Phase 7, seeding→Phase 8, enrichment→Phase 9 (M12.17); `init.yaml` `commit_message_language`→`commit_language` field-name fix with a contract regression test (M12.18); upgrade-mode prompt-default recovery via context-store (ADR 0042); dispatch-time observability fields `current_agent`/`current_task`/`iteration` (ADR 0041); lore-sync fired per-flow on `git push` via a PostToolUse hook (ADR 0043).
